> thread 方法是 app-server v2 管理 Codex conversation thread 的生命周期、持久化读取、命名、rollback、compaction、memory mode 和 raw item 注入的 client request catalog。

## 能回答的问题

- `thread/start`、`thread/resume`、`thread/fork` 的 params/response 类型是什么？
- `thread/list`、`thread/read`、`thread/turns/list` 支持哪些分页和过滤字段？
- `thread/rollback` 是否会回滚本地文件改动？
- 哪些 thread 方法是 experimental？
- `thread/inject_items` 与 `turn/start` 的差别是什么？

## 控制流与设计动机

thread 方法把“加载或新建 CodexThread”与“开始一个 turn”分离：`thread/start`、`thread/resume`、`thread/fork` 只返回 thread/runtime config，`turn/start` 才提交 user input。[I] 这种拆分让 IDE 可以先显示 thread、model、cwd、permission profile，再按用户动作发起 turn；`ThreadStartResponse` 中的 `permission_profile` 也明确被标为 canonical active permissions view。[E: codex-rs/app-server-protocol/src/protocol/v2.rs:3221]
