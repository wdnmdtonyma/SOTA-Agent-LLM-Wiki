> fs 与 command 方法是 app-server 暴露给客户端的 host filesystem、standalone process execution、fuzzy file search 控制面；它们不需要创建 thread 或 turn 就能读写文件、watch 文件变化或执行一次性命令。

## 能回答的问题

- `fs/readFile` 到 `fs/unwatch` 的 wire name、params type 和 response type 是什么？
- `command/exec` 为什么有 final response 和 `command/exec/outputDelta` notification 两条输出路径？
- `command/exec/write`、`command/exec/resize`、`command/exec/terminate` 如何用 `process_id` 定位进程？
- fuzzy file search 的一次性模式和 session 模式分别有哪些方法？

## 文件系统字段模型

## Standalone command 字段模型

`command/exec` 的 params 用 `command: Vec<String>` 表示 argv vector；`process_id` 是客户端可选提供的 connection-scoped id，但源码注释把它列为 `tty`、`streamStdin`、`streamStdoutStderr` 和 follow-up command 操作的必需条件。[E: codex-rs/app-server-protocol/src/protocol/v2.rs:2961][E: codex-rs/app-server-protocol/src/protocol/v2.rs:2963][E: codex-rs/app-server-protocol/src/protocol/v2.rs:2966][E: codex-rs/app-server-protocol/src/protocol/v2.rs:2967][E: codex-rs/app-server-protocol/src/protocol/v2.rs:2968][E: codex-rs/app-server-protocol/src/protocol/v2.rs:2971] `tty` 会隐含 stdin/stdout streaming；streamed stdout/stderr 不会复制到 final response；`output_bytes_cap` 与 `disable_output_cap`、`timeout_ms` 与 `disable_timeout` 是互斥配置。[E: codex-rs/app-server-protocol/src/protocol/v2.rs:2974][E: codex-rs/app-server-protocol/src/protocol/v2.rs:2976][E: codex-rs/app-server-protocol/src/protocol/v2.rs:2984][E: codex-rs/app-server-protocol/src/protocol/v2.rs:2985][E: codex-rs/app-server-protocol/src/protocol/v2.rs:2987][E: codex-rs/app-server-protocol/src/protocol/v2.rs:2990][E: codex-rs/app-server-protocol/src/protocol/v2.rs:2991][E: codex-rs/app-server-protocol/src/protocol/v2.rs:2994][E: codex-rs/app-server-protocol/src/protocol/v2.rs:2997][E: codex-rs/app-server-protocol/src/protocol/v2.rs:2999][E: codex-rs/app-server-protocol/src/protocol/v2.rs:3002][E: codex-rs/app-server-protocol/src/protocol/v2.rs:3004][E: codex-rs/app-server-protocol/src/protocol/v2.rs:3007][E: codex-rs/app-server-protocol/src/protocol/v2.rs:3008][E: codex-rs/app-server-protocol/src/protocol/v2.rs:3011]

`CommandExecResponse` 是退出后的 final buffered result，包含 `exit_code`、`stdout`、`stderr`；源码注释说明 streamed stdout/stderr 在 final response 中为空。[E: codex-rs/app-server-protocol/src/protocol/v2.rs:3038][E: codex-rs/app-server-protocol/src/protocol/v2.rs:3040][E: codex-rs/app-server-protocol/src/protocol/v2.rs:3043][E: codex-rs/app-server-protocol/src/protocol/v2.rs:3044][E: codex-rs/app-server-protocol/src/protocol/v2.rs:3047][E: codex-rs/app-server-protocol/src/protocol/v2.rs:3048]

## 控制流与设计动机

fs 方法提供“客户端直接操作宿主文件”的低延迟路径，command 方法提供“客户端直接操作进程”的低延迟路径，二者都避开 thread/turn 状态机。[I] `command/exec` 采用 outputDelta notification 加 final response 的双通道，是为了让 streaming clients 能逐块显示 stdout/stderr，同时仍能在进程退出时拿到 exit code 和非 streamed buffer。[I]
