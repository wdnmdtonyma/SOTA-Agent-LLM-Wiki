> config/account/model 方法是 app-server 面向客户端设置页、登录页、model picker、feature gate 和外部 agent 配置迁移 UI 的 client request catalog。

## 能回答的问题

- config read/write 方法如何表达 effective config、layers、version conflict 和 hot reload？
- account login 支持哪几种 login params variant？
- model list、experimental feature list、collaboration mode list 的分页/响应模型是什么？
- Windows sandbox setup、feedback upload 属于哪组 app-server client requests？

## Config 字段模型

`config/read` 的 params 有 `include_layers` 和可选 `cwd`；response 返回 effective `config`、每个 key 的 `origins` 和可选 `layers`。[E: codex-rs/app-server-protocol/src/protocol/v2.rs:872][E: codex-rs/app-server-protocol/src/protocol/v2.rs:874][E: codex-rs/app-server-protocol/src/protocol/v2.rs:879][E: codex-rs/app-server-protocol/src/protocol/v2.rs:885][E: codex-rs/app-server-protocol/src/protocol/v2.rs:887][E: codex-rs/app-server-protocol/src/protocol/v2.rs:888][E: codex-rs/app-server-protocol/src/protocol/v2.rs:890]

config write response 统一使用 `ConfigWriteResponse`，返回 `status`、`version`、canonical `file_path` 和可选 `overridden_metadata`；`config/value/write` 接收单个 key/value/merge strategy，`config/batchWrite` 接收 `edits` 并可 `reload_user_config` 到 loaded threads。[E: codex-rs/app-server-protocol/src/protocol/v2.rs:849][E: codex-rs/app-server-protocol/src/protocol/v2.rs:850][E: codex-rs/app-server-protocol/src/protocol/v2.rs:851][E: codex-rs/app-server-protocol/src/protocol/v2.rs:853][E: codex-rs/app-server-protocol/src/protocol/v2.rs:854][E: codex-rs/app-server-protocol/src/protocol/v2.rs:1056][E: codex-rs/app-server-protocol/src/protocol/v2.rs:1057][E: codex-rs/app-server-protocol/src/protocol/v2.rs:1058][E: codex-rs/app-server-protocol/src/protocol/v2.rs:1059][E: codex-rs/app-server-protocol/src/protocol/v2.rs:1070][E: codex-rs/app-server-protocol/src/protocol/v2.rs:1071][E: codex-rs/app-server-protocol/src/protocol/v2.rs:1079]

`configRequirements/read` response 里 `requirements` 可为 null；`ConfigRequirements` 覆盖 allowed approval policies、allowed approvals reviewers、allowed sandbox modes、allowed web search modes、feature requirements、residency 和 network requirements。[E: codex-rs/app-server-protocol/src/protocol/v2.rs:961][E: codex-rs/app-server-protocol/src/protocol/v2.rs:964][E: codex-rs/app-server-protocol/src/protocol/v2.rs:896][E: codex-rs/app-server-protocol/src/protocol/v2.rs:898][E: codex-rs/app-server-protocol/src/protocol/v2.rs:900][E: codex-rs/app-server-protocol/src/protocol/v2.rs:901][E: codex-rs/app-server-protocol/src/protocol/v2.rs:902][E: codex-rs/app-server-protocol/src/protocol/v2.rs:903][E: codex-rs/app-server-protocol/src/protocol/v2.rs:904][E: codex-rs/app-server-protocol/src/protocol/v2.rs:906]

## Account 字段模型

`LoginAccountParams` 是 tagged enum，wire `type` 可为 `apiKey`、`chatgpt`、`chatgptDeviceCode`、experimental `chatgptAuthTokens`；`LoginAccountResponse` 同样按 `type` 区分，ChatGPT OAuth 返回 `login_id` 和 `auth_url`，device code 返回 `login_id`、`verification_url`、`user_code`。[E: codex-rs/app-server-protocol/src/protocol/v2.rs:1895][E: codex-rs/app-server-protocol/src/protocol/v2.rs:1899][E: codex-rs/app-server-protocol/src/protocol/v2.rs:1906][E: codex-rs/app-server-protocol/src/protocol/v2.rs:1909][E: codex-rs/app-server-protocol/src/protocol/v2.rs:1914][E: codex-rs/app-server-protocol/src/protocol/v2.rs:1915][E: codex-rs/app-server-protocol/src/protocol/v2.rs:1933][E: codex-rs/app-server-protocol/src/protocol/v2.rs:1945][E: codex-rs/app-server-protocol/src/protocol/v2.rs:1947][E: codex-rs/app-server-protocol/src/protocol/v2.rs:1954][E: codex-rs/app-server-protocol/src/protocol/v2.rs:1956][E: codex-rs/app-server-protocol/src/protocol/v2.rs:1958]

`account/read` params 有 `refresh_token`，response 返回可选 `account` 和 `requires_openai_auth`；`account/rateLimits/read` response 同时保留 backward-compatible single bucket 和 keyed multi-bucket view。[E: codex-rs/app-server-protocol/src/protocol/v2.rs:2069][E: codex-rs/app-server-protocol/src/protocol/v2.rs:2076][E: codex-rs/app-server-protocol/src/protocol/v2.rs:2082][E: codex-rs/app-server-protocol/src/protocol/v2.rs:2083][E: codex-rs/app-server-protocol/src/protocol/v2.rs:2084][E: codex-rs/app-server-protocol/src/protocol/v2.rs:2029][E: codex-rs/app-server-protocol/src/protocol/v2.rs:2031][E: codex-rs/app-server-protocol/src/protocol/v2.rs:2033]

## 控制流与设计动机

config 方法把 read、single write、batch write、requirements read 分成独立 wire methods，允许 settings UI 在读取 effective state、处理 optimistic version conflict、展示 policy requirements 时调用不同端点。[I] account 方法把 login flow result 和 completion notification 拆开：`account/login/start` 返回 OAuth/device-code 操作入口，最终成功或失败由 `account/login/completed` notification 通知连接。[E: codex-rs/app-server-protocol/src/protocol/common.rs:1085][E: codex-rs/app-server-protocol/src/protocol/common.rs:1086][E: codex-rs/app-server-protocol/src/protocol/common.rs:1087][E: codex-rs/app-server-protocol/src/protocol/v2.rs:7188][E: codex-rs/app-server-protocol/src/protocol/v2.rs:7192][E: codex-rs/app-server-protocol/src/protocol/v2.rs:7193]
