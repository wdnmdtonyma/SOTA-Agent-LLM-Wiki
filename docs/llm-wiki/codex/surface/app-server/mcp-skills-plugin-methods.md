> MCP/skills/plugin 方法是 app-server 给客户端提供 skill discovery、marketplace/plugin 管理、apps/connectors 列表、MCP server inventory、MCP resource read/tool call 和 device key 操作的 client request catalog。

## 能回答的问题

- skills、marketplace、plugin、apps 的 app-server 方法如何分层？
- `mcpServerStatus/list`、`mcpServer/resource/read`、`mcpServer/tool/call` 的字段是什么？
- device key create/public/sign 三个方法分别返回什么？
- plugin install 为什么返回 `auth_policy` 和 `apps_needing_auth`？

## 控制流与设计动机

skills 和 plugins 共享“发现本地/远程能力，再由 config 写入启停状态”的控制面；MCP 共享“列 server inventory、读取 resource、直接 tool call、处理 OAuth login”的控制面。[I] device key 方法放在同一 catalog 页，是因为它们服务 remote-control/app connection identity，而不是普通 Codex turn 执行。[I]
