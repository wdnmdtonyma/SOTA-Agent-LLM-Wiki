## 设计动机与权衡

Realtime platform 将 native WebRTC 与 websocket protocol 分离，是因为 WebRTC path 依赖 macOS native module、platform audio device manager 和 worker runtime，而 websocket path 是 provider/base-url/header/session-update 驱动的 API transport。[I] 该结论由 macOS-only module cfg、native worker start/runtime、websocket client type/connect flow 共同支撑。[E: codex-rs/realtime-webrtc/src/lib.rs:1][E: codex-rs/realtime-webrtc/src/native.rs:60][E: codex-rs/realtime-webrtc/src/native.rs:83][E: codex-rs/realtime-webrtc/src/native.rs:137][E: codex-rs/codex-api/src/endpoint/realtime_websocket/methods.rs:552][E: codex-rs/codex-api/src/endpoint/realtime_websocket/methods.rs:560][E: codex-rs/codex-api/src/endpoint/realtime_websocket/methods.rs:637]

V2 conversational session 内建 `background_agent` 和 `remain_silent` tools，而 V2 transcription mode 没有 tools，说明 conversational realtime 被设计为可在语音/文本会话中委派后台 agent 或主动保持静默，而 transcription mode 只做输入转写。[I] 该结论由 V2 conversational/transcription 两个 session_update branches 支撑。[E: codex-rs/codex-api/src/endpoint/realtime_websocket/methods_v2.rs:105][E: codex-rs/codex-api/src/endpoint/realtime_websocket/methods_v2.rs:108][E: codex-rs/codex-api/src/endpoint/realtime_websocket/methods_v2.rs:124][E: codex-rs/codex-api/src/endpoint/realtime_websocket/methods_v2.rs:135][E: codex-rs/codex-api/src/endpoint/realtime_websocket/methods_v2.rs:153][E: codex-rs/codex-api/src/endpoint/realtime_websocket/methods_v2.rs:155]
