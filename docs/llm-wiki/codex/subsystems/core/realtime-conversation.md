> Realtime conversation is a side-channel subsystem beside normal turns: `Op::RealtimeConversationStart/Audio/Text/Speech/Close/ListVoices` are dispatched by `session/handlers.rs`, `RealtimeConversationManager` owns the active realtime state, and realtime outputs include lifecycle/streaming/SDP/list-voices `EventMsg::RealtimeConversation*` payloads.[E: codex-rs/core/src/session/handlers.rs:71][E: codex-rs/core/src/session/handlers.rs:718][E: codex-rs/core/src/session/handlers.rs:733][E: codex-rs/core/src/session/handlers.rs:737][E: codex-rs/core/src/session/handlers.rs:741][E: codex-rs/core/src/session/handlers.rs:745][E: codex-rs/core/src/session/handlers.rs:749][E: codex-rs/core/src/realtime_conversation.rs:96][E: codex-rs/protocol/src/protocol.rs:1230][E: codex-rs/protocol/src/protocol.rs:1233][E: codex-rs/protocol/src/protocol.rs:1236][E: codex-rs/protocol/src/protocol.rs:1239][E: codex-rs/protocol/src/protocol.rs:1365]

## 能回答的问题

- Realtime start/audio/text/speech/close/list voices 分别走哪个 handler?

## Start path

The dispatch loop routes realtime start/audio/text/speech/close/list-voices ops to realtime handlers and returns without starting a normal `RegularTask`.[E: codex-rs/core/src/session/handlers.rs:718][E: codex-rs/core/src/session/handlers.rs:720][E: codex-rs/core/src/session/handlers.rs:731][E: codex-rs/core/src/session/handlers.rs:733][E: codex-rs/core/src/session/handlers.rs:738][E: codex-rs/core/src/session/handlers.rs:742][E: codex-rs/core/src/session/handlers.rs:746][E: codex-rs/core/src/session/handlers.rs:750]

## Input and handoff path

`audio_in()` writes audio frames into the active channel, dropping frames when the queue is full; `text_in()` requires a running conversation, prefixes user text for V2, and sends it to the text channel.[E: codex-rs/core/src/realtime_conversation.rs:447][E: codex-rs/core/src/realtime_conversation.rs:453][E: codex-rs/core/src/realtime_conversation.rs:459][E: codex-rs/core/src/realtime_conversation.rs:461][E: codex-rs/core/src/realtime_conversation.rs:471][E: codex-rs/core/src/realtime_conversation.rs:479][E: codex-rs/core/src/realtime_conversation.rs:485][E: codex-rs/core/src/realtime_conversation.rs:487][E: codex-rs/core/src/realtime_conversation.rs:489]

`handoff_out()` converts Codex output into `RealtimeOutbound`: it can no-op when client-managed handoffs are enabled, update/append an active handoff, create conversation items, or send a standalone handoff; `handoff_complete()` is V2-only and either acknowledges or completes the active handoff depending on response-as-items mode.[E: codex-rs/core/src/realtime_conversation.rs:496][E: codex-rs/core/src/realtime_conversation.rs:511][E: codex-rs/core/src/realtime_conversation.rs:519][E: codex-rs/core/src/realtime_conversation.rs:523][E: codex-rs/core/src/realtime_conversation.rs:530][E: codex-rs/core/src/realtime_conversation.rs:541][E: codex-rs/core/src/realtime_conversation.rs:547][E: codex-rs/core/src/realtime_conversation.rs:558][E: codex-rs/core/src/realtime_conversation.rs:601][E: codex-rs/core/src/realtime_conversation.rs:609][E: codex-rs/core/src/realtime_conversation.rs:612][E: codex-rs/core/src/realtime_conversation.rs:624][E: codex-rs/core/src/realtime_conversation.rs:627]
