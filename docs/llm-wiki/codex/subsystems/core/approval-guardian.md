> Guardian 自动审批审查是在 `OnRequest` 或 `Granular(_)` approval 场景下，用独立 locked-down Codex subagent 审核具体 planned action，并返回 allow/deny structured JSON 的 automated reviewer；timeout 返回 `TimedOut`，session/prompt/parse failure 会构造 high-risk deny assessment。[E: codex-rs/core/src/guardian/review.rs:146][E: codex-rs/core/src/guardian/review.rs:147][E: codex-rs/core/src/guardian/review_session.rs:827][E: codex-rs/core/src/guardian/review_session.rs:839][E: codex-rs/core/src/guardian/review_session.rs:841][E: codex-rs/core/src/guardian/review_session.rs:873][E: codex-rs/core/src/guardian/prompt.rs:571][E: codex-rs/core/src/guardian/review.rs:329][E: codex-rs/core/src/guardian/review.rs:361][E: codex-rs/core/src/guardian/review.rs:363][E: codex-rs/core/src/guardian/review.rs:385][E: codex-rs/core/src/guardian/review.rs:387]

## 职责边界

`guardian` 模块负责把 approval request 格式化为 action JSON、构建审查 prompt、运行 review session、解析 structured output、发 `GuardianAssessment` events、缓存 deny rationale；被审核 action 的实际执行仍由 tool runtime / orchestrator 在 approval phase 之后继续。[E: codex-rs/core/src/guardian/approval_request.rs:233][E: codex-rs/core/src/guardian/prompt.rs:92][E: codex-rs/core/src/guardian/prompt.rs:98][E: codex-rs/core/src/guardian/review.rs:252][E: codex-rs/core/src/guardian/review.rs:424][E: codex-rs/core/src/guardian/review.rs:430][E: codex-rs/core/src/tools/orchestrator.rs:188][E: codex-rs/core/src/tools/orchestrator.rs:220][E: codex-rs/core/src/tools/orchestrator.rs:227][I] Guardian 是否启用由 `routes_approval_to_guardian` 判定：approval policy 必须是 `OnRequest` 或 `Granular(_)`，且 config 的 `approvals_reviewer` 必须是 `GuardianSubagent`。[E: codex-rs/core/src/guardian/review.rs:146][E: codex-rs/core/src/guardian/review.rs:147]

## 设计动机与权衡

Guardian 使用独立 Codex subagent 而不是在 parent turn 内直接问同一个 model，是为了让审查拥有单独的 policy prompt、read-only sandbox、Never approval 和清空后的 MCP/app/plugin 能力面；这降低了审查过程本身产生副作用或递归审批的风险。[E: codex-rs/core/src/guardian/review_session.rs:831][E: codex-rs/core/src/guardian/review_session.rs:839][E: codex-rs/core/src/guardian/review_session.rs:841][E: codex-rs/core/src/guardian/review_session.rs:845][E: codex-rs/core/src/guardian/review_session.rs:868][E: codex-rs/core/src/guardian/review_session.rs:869][E: codex-rs/core/src/guardian/review_session.rs:873][I]

Full/Delta prompt 与 reusable trunk 的组合是在 prompt-cache 稳定性和 parallel approvals latency 之间折中：空闲 trunk 复用上下文，busy trunk 走 ephemeral fork，避免并行审批互相阻塞或污染 trunk state。[E: codex-rs/core/src/guardian/review_session.rs:367][E: codex-rs/core/src/guardian/review_session.rs:370][E: codex-rs/core/src/guardian/review_session.rs:601][E: codex-rs/core/src/guardian/review_session.rs:602][E: codex-rs/core/src/guardian/review_session.rs:604][I]

Fail-closed 策略把 timeout、session failure、parse failure 都视作不允许继续执行；但 timeout message 明确区分“审查未完成”和“动作不安全”，调用方可选择重试或询问用户。[E: codex-rs/core/src/guardian/review.rs:289][E: codex-rs/core/src/guardian/review.rs:329][E: codex-rs/core/src/guardian/review.rs:361][E: codex-rs/core/src/guardian/review.rs:363][E: codex-rs/core/src/guardian/review.rs:52][E: codex-rs/core/src/guardian/review.rs:54][I]

## gotcha

- Guardian routing gate 只看 approval policy 与 `approvals_reviewer`；如果 exec policy 在更早阶段直接 `Forbidden`，Guardian 不会看到该 action。[E: codex-rs/core/src/tools/orchestrator.rs:136][E: codex-rs/core/src/tools/orchestrator.rs:137][E: codex-rs/core/src/guardian/review.rs:144][E: codex-rs/core/src/guardian/review.rs:147]
- Guardian output schema 只强制 `outcome`，因此 downstream 不能假设 model 一定显式返回 risk/user authorization/rationale；parser 会补默认值。[E: codex-rs/core/src/guardian/prompt.rs:571][E: codex-rs/core/src/guardian/prompt.rs:511][E: codex-rs/core/src/guardian/prompt.rs:518][E: codex-rs/core/src/guardian/prompt.rs:524][E: codex-rs/core/src/guardian/prompt.rs:531]
- Deny rationale 通过 review id 存在 `guardian_rejections`，`guardian_rejection_message` 读取后会 remove；重复读取同一 review id 可能回退到默认 rationale。[E: codex-rs/core/src/guardian/review.rs:62][E: codex-rs/core/src/guardian/review.rs:67][E: codex-rs/core/src/guardian/review.rs:68][E: codex-rs/core/src/guardian/review.rs:69]
- Guardian transcript 明确跳过 `is_contextual_user_message_content(content)` 为真的 user messages；具体哪些 fragment 命中该 helper 由 prompt/context helper 实现决定。[E: codex-rs/core/src/guardian/prompt.rs:357][E: codex-rs/core/src/guardian/prompt.rs:358][E: codex-rs/core/src/guardian/prompt.rs:359][I]
