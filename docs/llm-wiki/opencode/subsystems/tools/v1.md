> V1 工具系统是当前活跑主线：`SessionPrompt.runLoop` 调 `SessionTools.resolve`，V1 registry 装配内建/插件/MCP 工具，最终以 Vercel AI SDK tool shape 交给 `streamText`。

## 能回答的问题

- V1 的内建工具、filesystem plugin tools、`plugin.list()` tools 和 MCP tools 分别从哪里进入模型可见工具集。
- `apply_patch` 为什么只在 GPT patch 条件下暴露，`edit/write` 为什么会被隐藏。
- V1 工具执行时 `ctx.ask`、plugin hooks、attachments、output truncation 和 AI SDK bridge 怎样串起来。
- `packages/llm` 在 V1 里为什么只是 `OPENCODE_EXPERIMENTAL_NATIVE_LLM` seam，而不是 V1 默认工具执行路径。

## 1 职责与边界

### 职责

- 维护 V1 工具 registry service，提供 `ids()`, `all()`, `named()`, `tools(model)` 四个接口。[E: packages/opencode/src/tool/registry.ts:70]
- 装配内建工具、filesystem-discovered plugin tools、runtime plugin tools，并按 provider/model 做 tool list 过滤；TaskTool 的 agent permissions 会改写 task tool 描述中的可用 agent 列表。[E: packages/opencode/src/tool/registry.ts:174][E: packages/opencode/src/tool/registry.ts:188][E: packages/opencode/src/tool/registry.ts:267][E: packages/opencode/src/tool/registry.ts:255]
- 把 V1 `Tool.Def` 转成 AI SDK `tool({ description, inputSchema, execute })`，并在 execute 前后触发 plugin hooks。[E: packages/opencode/src/session/tools.ts:80][E: packages/opencode/src/session/tools.ts:87][E: packages/opencode/src/session/tools.ts:102]
- 把 MCP tools 注入最终 AI SDK `tools` map，并对 MCP 文本输出做 V1 truncation。[E: packages/opencode/src/session/tools.ts:117][E: packages/opencode/src/session/tools.ts:176][E: packages/opencode/src/session/tools.ts:201]

### 非职责

- V1 registry 不负责 provider stream 本身；默认 stream 由 `LLM.stream` 内部的 AI SDK `streamText` 发起。[E: packages/opencode/src/session/llm.ts:280][E: packages/opencode/src/session/llm.ts:279]
- V1 registry 不负责 V2 durable settlement [I]；V1 AI SDK 甚至可能在 start-step event 前内部执行工具，processor 因此要预先捕获 snapshot。[E: packages/opencode/src/session/processor.ts:114]
- V1 registry 不是旧 `Bus` service。当前源码暴露的是 `GlobalBus` emitter，EventV2Bridge 把 EventV2 重新 emit 到 GlobalBus；`sync/README.md` 是关于早期 Bus/sync 设计的说明文档。[E: packages/opencode/src/bus/global.ts:22][E: packages/opencode/src/event-v2-bridge.ts:42][E: packages/opencode/src/sync/README.md:35]

## 4 控制流

1. `SessionPrompt.runLoop` 在 provider call 前调用 `SessionTools.resolve`，并注入 `Plugin`, `Permission`, `ToolRegistry`, `MCP`, `Truncate` services。[E: packages/opencode/src/session/prompt.ts:1279][E: packages/opencode/src/session/prompt.ts:1288]
2. `ToolRegistry.layer` 读取 config、plugin、agent、truncate、runtime flags，并初始化各个内建工具 Effect service。[E: packages/opencode/src/tool/registry.ts:86][E: packages/opencode/src/tool/registry.ts:92]
3. filesystem plugin tools 通过 `Glob.scanSync("{tool,tools}/*.{js,ts}")` 发现，动态 import 后把 `default` export 命名为文件 namespace，其它 export 命名为 `${namespace}_${id}`。[E: packages/opencode/src/tool/registry.ts:174][E: packages/opencode/src/tool/registry.ts:181][E: packages/opencode/src/tool/registry.ts:184]
4. runtime plugins 通过 `plugin.list()` 注入 `p.tool` 里的 definitions。[E: packages/opencode/src/tool/registry.ts:188][E: packages/opencode/src/tool/registry.ts:190]
5. registry 初始化 16 个内建 tool slots：invalid, shell, read, glob, grep, edit, write, task, fetch, todo, search, skill, patch, question, lsp, plan；实际 builtin list 会按 client/flags 条件包含 question/lsp/plan。[E: packages/opencode/src/tool/registry.ts:198][E: packages/opencode/src/tool/registry.ts:219][E: packages/opencode/src/tool/registry.ts:234]
6. `tools(model)` 先按 websearch 可用性、GPT patch gating 过滤工具列表；TaskTool 另按 task agent permissions 过滤并改写 task tool 描述中的 agent 列表。[E: packages/opencode/src/tool/registry.ts:267][E: packages/opencode/src/tool/registry.ts:269][E: packages/opencode/src/tool/registry.ts:274][E: packages/opencode/src/tool/registry.ts:252][E: packages/opencode/src/tool/registry.ts:255]
7. 每个 registry tool 经过 `plugin.trigger("tool.definition")`，插件可以改 description/parameters/jsonSchema。[E: packages/opencode/src/tool/registry.ts:281][E: packages/opencode/src/tool/registry.ts:289]
8. `SessionTools.resolve` 把 registry tools 转成 AI SDK `tool(...)`，其中 input schema 来自 provider transform 后的 schema。[E: packages/opencode/src/session/tools.ts:74][E: packages/opencode/src/session/tools.ts:79][E: packages/opencode/src/session/tools.ts:80]
9. AI SDK 执行工具时，V1 bridge 触发 `tool.execute.before`，调用 tool execute，附加 file parts，再触发 `tool.execute.after`。[E: packages/opencode/src/session/tools.ts:87][E: packages/opencode/src/session/tools.ts:95][E: packages/opencode/src/session/tools.ts:102]
10. MCP tools 在 registry tools 之外注入；MCP bridge 使用 `mcp.tools()` map key 作为 permission key，文本结果会经 `truncate.output` 裁剪，非文本 attachments 保留。[E: packages/opencode/src/session/tools.ts:117][E: packages/opencode/src/session/tools.ts:134][E: packages/opencode/src/session/tools.ts:176][E: packages/opencode/src/session/tools.ts:187][E: packages/opencode/src/session/tools.ts:201]
11. `handle.process` 接收 `tools`，再走 `LLM.stream`；默认 path 是 AI SDK `streamText`，`activeTools` 和 `tools` 都来自 `prepared.tools`。[E: packages/opencode/src/session/prompt.ts:1336][E: packages/opencode/src/session/prompt.ts:1344][E: packages/opencode/src/session/processor.ts:974][E: packages/opencode/src/session/llm.ts:317][E: packages/opencode/src/session/llm.ts:318]

## 5 设计动机与 tradeoff

- V1 registry 同时承担 discovery、filter、prompt-shaping 和 execution bridge，是 AI SDK era 的高耦合中枢；这解释了为什么 tool-specific behavior 分散在 registry、session/tools 和每个 tool 文件中。[I]
- GPT patch gating 是 V1 的模型策略：`modelID.includes("gpt-") && !includes("oss") && !includes("gpt-4")` 时暴露 `apply_patch`，并隐藏 `edit/write`。[E: packages/opencode/src/tool/registry.ts:274][E: packages/opencode/src/tool/registry.ts:275][E: packages/opencode/src/tool/registry.ts:276]
- V1 plugin compatibility 优先：filesystem plugin tools 仍支持 Zod args 和 legacy JSON schema conversion，registry 注释说明 pre-1.14.49 曾依赖 `z.object(def.args)` 容忍 undefined。[E: packages/opencode/src/tool/registry.ts:114][E: packages/opencode/src/tool/registry.ts:119][E: packages/opencode/src/tool/registry.ts:123]
- V1 默认不是 native provider protocol runtime。`OPENCODE_EXPERIMENTAL_NATIVE_LLM` 打开时会尝试 native seam；不支持时明确 fallback 到 AI SDK。[E: packages/opencode/src/effect/runtime-flags.ts:53][E: packages/opencode/src/session/llm.ts:226][E: packages/opencode/src/session/llm.ts:260]
- 即使走 native seam，工具执行仍是 opencode-owned，并把 native tool call 适配回 AI SDK Tool.execute shape。[E: packages/opencode/src/session/llm/native-runtime.ts:175][E: packages/opencode/src/session/llm/native-runtime.ts:181]

## 6 Gotchas

- `packages/opencode/src/session/message-v2.ts` 名字里的 `v2` 不代表 V2 core tool runtime；V1 run loop 在这里调用的是 `MessageV2.toModelMessagesEffect` 做 V1 到 AI SDK/model message 的转换。[E: packages/opencode/src/session/prompt.ts:1331][I]
- V1 的 shell 工具文件名是 `shell.ts`，但 wire id 和 permission key 是 `bash`，这是 plugin/user/saved permissions 兼容要求。[E: packages/opencode/src/tool/shell/id.ts:16][E: packages/opencode/src/tool/shell/id.ts:17]
- V1 tools 可以来自 MCP，也可以来自 plugin；registry `ids()` 只覆盖 registry tools，不代表最终 AI SDK tools map 中没有 MCP entries。[E: packages/opencode/src/tool/registry.ts:248][E: packages/opencode/src/session/tools.ts:117]
- `sync/README.md` 讲的是早期 Bus/sync 兼容设计；当前可检索代码里的 EventV2 listener/re-emit bridge 会把事件 emit 到 `GlobalBus`。[E: packages/opencode/src/sync/README.md:39][E: packages/opencode/src/event-v2-bridge.ts:42][E: packages/opencode/src/event-v2-bridge.ts:52]
