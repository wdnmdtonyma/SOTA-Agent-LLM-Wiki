> V2 工具系统是 `@opencode/v2` core 的 opaque tool carrier：`Tool.make` 捕获 runtime，scoped registry 在 provider turn 前 materialize definitions，tool call 到达后 settle 并统一 output bounding。

## 能回答的问题

- V2 的 tool value 为什么没有内建 name，name 为什么来自 registration record key。
- Application tools、Location builtins、future plugin/MCP tools 如何共用 `Tool.make` contract。
- latest-wins scoped registry、stale rejection、materialize/settle 怎样保证 provider turn 一致性。
- V2 runner 如何在一次 provider turn 中执行 local tool call 并发布 durable tool result。

## 1 职责与边界

### 职责

- 提供 opaque `Tool.Definition<Input, Output>`；`Tool.make` 创建 frozen empty object，并把 derived definition/settle runtime 存入 WeakMap。[E: packages/core/src/tool/tool.ts:20][E: packages/core/src/tool/tool.ts:60][E: packages/core/src/tool/tool.ts:65][E: packages/core/src/tool/tool.ts:67]
- 提供 narrow `Tools.Service.register` 给 builtins/plugin/application 层，而不是暴露内部 registry。[E: packages/core/src/tool/tools.ts:6][E: packages/core/src/tool/registry.ts:128]
- 在 Location-scoped registry 中合并 application tools 和 local tools，执行 latest-wins 规则，并在 provider turn materialize definitions。[E: packages/core/src/tool/registry.ts:44][E: packages/core/src/tool/registry.ts:105]
- settle tool call：registry 校验 stale/unknown 并调用 runtime settle；`Tool.settle` 负责 decode input、执行 tool、encode output、projection；registry 再做 output bounding 并返回 result/output/outputPaths。[E: packages/core/src/tool/registry.ts:49][E: packages/core/src/tool/registry.ts:61][E: packages/core/src/tool/tool.ts:81][E: packages/core/src/tool/tool.ts:84][E: packages/core/src/tool/tool.ts:86][E: packages/core/src/tool/tool.ts:95][E: packages/core/src/tool/tool.ts:98][E: packages/core/src/tool/registry.ts:74]

### 非职责

- Registry 不注入 permission helper；trusted tools 自己捕获 `PermissionV2.Service` 并按 action/resource 申请权限。[E: specs/v2/tools.md:131]
- V2 tool bounding 不是 producer memory management；spec 明确 producer/streaming source 需要自己的 capture/spooling limits。[E: specs/v2/tools.md:159]
- V2 core 目前真正接通执行的公共入口是 embedded Effect API `packages/core/src/public/opencode.ts`，该 API 暴露 `tools.register` 和 `sessions.*`。[E: packages/core/src/public/opencode.ts:24][E: packages/core/src/public/opencode.ts:90]

## 4 控制流

1. V2 tool author 调用 `Tool.make({ description, input, output, execute, toModelOutput })`；runtime 存入 WeakMap，tool value 本身是 frozen empty object。[E: packages/core/src/tool/tool.ts:62][E: packages/core/src/tool/tool.ts:65][E: packages/core/src/tool/tool.ts:67]
2. Builtins 在 Location layer 中调用各自 `*.layer` 注册工具；`builtins.ts` 注释说明这里只合并 shipped Location-scoped built-ins，MCP/plugin tools later use separate scoped registrations。[E: packages/core/src/tool/builtins.ts:31][E: packages/core/src/tool/builtins.ts:32]
3. `Tools.Service` 只暴露 `register`，它由 internal `ToolRegistry.Service.register` 支撑。[E: packages/core/src/tool/tools.ts:6][E: packages/core/src/tool/registry.ts:128]
4. registry register 校验 name，写入 local map；Scope finalizer 只移除本次 token，reveals previous registration。[E: packages/core/src/tool/registry.ts:87][E: packages/core/src/tool/registry.ts:90][E: packages/core/src/tool/registry.ts:96]
5. provider turn 前，runner 调用 `tools.materialize(agent.info?.permissions)`。[E: packages/core/src/session/runner/llm.ts:217]
6. materialize 先复制 application entries，再用 local latest registrations 覆盖；被 permission rules wholly disabled 的 tool 被删除。[E: packages/core/src/tool/registry.ts:105][E: packages/core/src/tool/registry.ts:107][E: packages/core/src/tool/registry.ts:111]
7. materialize 返回 model-facing definitions；runner 把它们放进 `LLM.request({ tools })`。[E: packages/core/src/tool/registry.ts:114][E: packages/core/src/session/runner/llm.ts:219][E: packages/core/src/session/runner/llm.ts:226]
8. runner 调用 exactly one `llm.stream(request)` for each provider turn，注释明确 tool execution 和 continuation 留在 runner 内。[E: packages/core/src/session/runner/llm.ts:245][E: packages/core/src/session/runner/llm.ts:246]
9. provider stream 产生 local `tool-call` 且不是 providerExecuted 时，runner 获取 assistantMessageID 并调用 materialized `settle`。[E: packages/core/src/session/runner/llm.ts:256][E: packages/core/src/session/runner/llm.ts:258][E: packages/core/src/session/runner/llm.ts:261]
10. registry settle 校验 unknown/stale，再调用 tool runtime settle，捕获 `LLM.ToolFailure` 为 model-visible error result。[E: packages/core/src/tool/registry.ts:52][E: packages/core/src/tool/registry.ts:59][E: packages/core/src/tool/registry.ts:68]
11. 成功 output 经 `ToolOutputStore.bound` 后转换为 `ToolResultValue`，outputPaths 随 settlement 返回 runner。[E: packages/core/src/tool/registry.ts:74][E: packages/core/src/tool/registry.ts:75]
12. runner 发布 `LLMEvent.toolResult`，携带 result/output，并把 outputPaths 交给 publisher。[E: packages/core/src/session/runner/llm.ts:268][E: packages/core/src/session/runner/llm.ts:270][E: packages/core/src/session/runner/llm.ts:276]

## 5 设计动机与 tradeoff

- V2 把 tool name 从 tool value 中移出：spec 明确 record key 是 model-facing name，可重用 tool value 本身没有 intrinsic name。[E: specs/v2/tools.md:67]
- Opaque carrier 让 registry 可以私有派生 provider definition 和解释 invocation，调用方依赖 `Tool.make` inference，而不是拿到 executor/schema public fields。[E: specs/v2/tools.md:31]
- Scoped registration 规则保证 overlay 可逆：latest active registration wins，closing winner reveals next-latest active registration。[E: specs/v2/tools.md:83][E: packages/core/src/tool/registry.ts:96]
- Materialization 捕获 advertised registration identity；settlement 时如果同名注册被替换或移除，返回 stale tool call，而不是执行新 handler。[E: specs/v2/tools.md:151][E: packages/core/src/tool/registry.ts:59]
- V2 runner 在 `uninterruptibleMask` 中先 `restore(toolMaterialization.settle(...))`，settlement 本身可被 restore interrupt；settlement 完成后，发布 `LLMEvent.toolResult` 与 outputPaths 发生在同一个 masked continuation 内。[E: packages/core/src/session/runner/llm.ts:259][E: packages/core/src/session/runner/llm.ts:260][E: packages/core/src/session/runner/llm.ts:268][E: packages/core/src/session/runner/llm.ts:276]

## 6 Gotchas

- V2 builtins static list 仍有 launch-follow-up TODO：edit fuzzy parity、task、LSP、plan_exit 等被明确列为待移植项。[E: packages/core/src/tool/builtins.ts:31]
- V2 Bash 已接通但 spec 明确“不 sandboxed”；external workdir approval enforced，command-arg absolute warnings advisory。[E: specs/v2/session.md:194]
- `packages/core/src/connector.ts` 名字容易误读；本批工具系统证据里没有把它作为云连接器参与 tool registry 流程。[I]
- V2 public `OpenCode.Service` 是 Effect applications embedding OpenCode 的 intentional native API，当前公共层暴露 `tools.register` 和 `sessions.*`。[E: packages/core/src/public/opencode.ts:24][E: packages/core/src/public/opencode.ts:90]
