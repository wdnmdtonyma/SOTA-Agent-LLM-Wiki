> `ref.ai.image-models` 是 pi-ai 图像模型 catalog: 以 generated `IMAGE_MODELS` 为 ground truth,逐实例列出 OpenRouter 图像模型的 id/name、provider、能力字段、成本字段和 provider 装配关系。

## 能回答的问题

- `IMAGE_MODELS` 当前包含哪些 image model 实例?
- 每个 image model 的 `api`、`provider`、`baseUrl`、`input`、`output`、`cost` 字段是什么?
- 哪些图像模型输出同时声明 text 和 image?
- OpenRouter image provider 如何消费 `IMAGE_MODELS.openrouter`?
- 为什么不要直接手改 `image-models.generated.ts`?

## Ground Truth

`IMAGE_MODELS` 由 `packages/ai/src/image-models.generated.ts` 导出,当前顶层只有 `openrouter` provider bucket [E: packages/ai/src/image-models.generated.ts:6] [E: packages/ai/src/image-models.generated.ts:7]。文件尾部把整体对象约束为 `Record<string, Record<string, ImagesModel<ImagesApi>>>`,所以本目录把每个二级 key 视为一个 image model 实例 [E: packages/ai/src/image-models.generated.ts:564] [I]。

`openrouterImagesProvider()` 构造 `ImagesProvider`: provider id 是 `openrouter`,display name 是 `OpenRouter`,认证读取 `OPENROUTER_API_KEY`,models 来自 `Object.values(IMAGE_MODELS.openrouter)`,api implementation 来自 `openrouterImagesApi()` [E: packages/ai/src/providers/openrouter-images.ts:6] [E: packages/ai/src/providers/openrouter-images.ts:7] [E: packages/ai/src/providers/openrouter-images.ts:8] [E: packages/ai/src/providers/openrouter-images.ts:9] [E: packages/ai/src/providers/openrouter-images.ts:10] [E: packages/ai/src/providers/openrouter-images.ts:11] [E: packages/ai/src/providers/openrouter-images.ts:12]。因此表中每个 `IMAGE_MODELS.openrouter` 条目都会进入 OpenRouter image provider 的 static model list [E: packages/ai/src/providers/openrouter-images.ts:11] [I]。

## Generated Gotcha

`image-models.generated.ts` 的文件头声明自己由 `scripts/generate-image-models.ts` 自动生成,并提示不要手工编辑,而是运行 `npm run generate-image-models` 更新 [E: packages/ai/src/image-models.generated.ts]。本节点可以引用 generated catalog 中的事实,但若模型列表需要变更,应追到生成脚本或上游数据,不要把 generated 文件当手写配置入口 [I]。
