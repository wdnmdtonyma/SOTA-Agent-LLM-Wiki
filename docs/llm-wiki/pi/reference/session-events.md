> `ref.coding-agent.session-events` 是 coding-agent 产品层 `AgentSessionEvent` 的逐实例 catalog:以 `packages/coding-agent/src/core/agent-session.ts` 中的 `AgentSessionEvent` union、`_handleAgentEvent()`、`_emitExtensionEvent()` 和 session-owned `_emit()` call site 为 ground truth。

## 能回答的问题

- `AgentSessionEvent` 当前有哪些顶层 `type` discriminator?
- coding-agent 如何继承 core `AgentEvent`,又如何替换 `agent_end`?
- 哪些事件是 `AgentSession` 自己定义和发射的 session-owned event?
- 哪些 inherited event 的精确 payload shape 需要回到 `ref.agent.agent-events` 核?
- `willRetry`、compaction、thinking level 和 retry 事件分别在哪些代码路径发出?

## AgentSessionEvent 边界

`AgentSessionEvent` 在 `agent-session.ts` 中定义为三段:第一段是 `Exclude<AgentEvent, { type: "agent_end" }>`;第二段是 coding-agent replacement `agent_end` 并额外加入 `willRetry: boolean`;第三段是 `queue_update`、`compaction_start`、`session_info_changed`、`thinking_level_changed`、`compaction_end`、`auto_retry_start` 和 `auto_retry_end` 7 个 session-owned events。[E: packages/coding-agent/src/core/agent-session.ts:127][E: packages/coding-agent/src/core/agent-session.ts:128][E: packages/coding-agent/src/core/agent-session.ts:129][E: packages/coding-agent/src/core/agent-session.ts:135][E: packages/coding-agent/src/core/agent-session.ts:151]

`_handleAgentEvent()` 对 core `agent_end` 以 `{ ...event, willRetry: this._willRetryAfterAgentEnd(event) }` 形式转成 session event,对其它 inherited `AgentEvent` 原样转发给 session listeners; inherited payload 的完整静态 union 由 [ref.agent.agent-events](agent-events.md) 覆盖。[E: packages/coding-agent/src/core/agent-session.ts:541][I]

## 传输与持久化边界

`AgentSession` constructor 始终订阅底层 `agent` event;对外 `subscribe(listener)` 保存 `AgentSessionEventListener` 并返回 unsubscribe function。[E: packages/coding-agent/src/core/agent-session.ts:355][E: packages/coding-agent/src/core/agent-session.ts:718][E: packages/coding-agent/src/core/agent-session.ts:719][E: packages/coding-agent/src/core/agent-session.ts:722]

`_emit(event)` 是 session listeners 的 fan-out helper;`_handleAgentEvent()` 在 extension event 后把 inherited events 或 replacement `agent_end` 发给 listeners,而 session-owned events 由 `AgentSession` 方法直接调用 `_emit()`。[E: packages/coding-agent/src/core/agent-session.ts:496][E: packages/coding-agent/src/core/agent-session.ts:497][E: packages/coding-agent/src/core/agent-session.ts:498][E: packages/coding-agent/src/core/agent-session.ts:541][E: packages/coding-agent/src/core/agent-session.ts:503][E: packages/coding-agent/src/core/agent-session.ts:1585][E: packages/coding-agent/src/core/agent-session.ts:2716]

JSON mode、RPC stdout、session header、RPC-only records、docs snippet freshness 和 client-side stdout casts 不在本节点的 `index.json` source 中;这些输出面 contract 应由 `ref.coding-agent.json-events`、`surface.modes.print` 或 `surface.modes.rpc-protocol` 覆盖。[I]
