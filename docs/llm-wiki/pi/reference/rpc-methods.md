> `ref.coding-agent.rpc-methods` 是 pi coding-agent RPC stdin command catalog:以 `RpcCommand` union 为方法全集,并用 `rpc-mode.ts` 的 switch dispatch 和 `docs/rpc.md` 的用户协议说明逐命令核对输入字段、成功 response 和运行时行为。

## 能回答的问题

- pi RPC mode 当前支持哪些 stdin command?
- 每个 RPC command 的输入字段、可选字段和成功 response payload 是什么?
- `prompt`、`steer`、`follow_up` 三种发送消息入口有什么差异?
- session/model/thinking/queue/compaction/retry/bash 相关 command 分别调用哪个 runtime 方法?
- `get_commands` 返回的是内置 slash command 还是 extension/prompt/skill command?
- `extension_ui_response` 和 extension UI request 是否计入 `RpcCommand` 方法目录?

## Response 与文档差异

`RpcResponse` 的 success variants 与 29 个 `RpcCommand` command 名一一对应,并额外有通用 failure variant;这说明 parse/unknown/validation/runtime failure 走同一 response envelope,而不是每个 command 单独定义 failure union [E: packages/coding-agent/src/modes/rpc/rpc-types.ts:111] [E: packages/coding-agent/src/modes/rpc/rpc-types.ts:113] [E: packages/coding-agent/src/modes/rpc/rpc-types.ts:202] [E: packages/coding-agent/src/modes/rpc/rpc-types.ts:206] [I]。

`docs/rpc.md` 的 `get_commands` 示例和说明列出 `location`、`path` 等字段,但当前 `RpcSlashCommand` 类型和 `rpc-mode.ts` dispatch 实际输出 `sourceInfo` 对象字段,不是 top-level `location`/`path` 字段 [E: packages/coding-agent/docs/rpc.md:720] [E: packages/coding-agent/docs/rpc.md:736] [E: packages/coding-agent/docs/rpc.md:740] [E: packages/coding-agent/src/modes/rpc/rpc-types.ts:84] [E: packages/coding-agent/src/modes/rpc/rpc-mode.ts:643] [E: packages/coding-agent/src/modes/rpc/rpc-mode.ts:652] [E: packages/coding-agent/src/modes/rpc/rpc-mode.ts:661] [U]。
