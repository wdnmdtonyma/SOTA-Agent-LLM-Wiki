> `ref.tools-catalog` 是 pi coding-agent 内置工具集的逐实例目录:以 `packages/coding-agent/src/core/tools/index.ts` 为 ground truth,列出 wire name、definition factory、preset、输入、输出 details、装配路径和源文件。

## 能回答的问题

- pi 内置工具的完整 wire name 清单是什么?
- `createCodingToolDefinitions`、`createReadOnlyToolDefinitions`、`createAllToolDefinitions` 分别包含哪些工具?
- 每个内置工具的 TypeBox 输入 schema 和 result `details` 大致是什么?
- built-in `ToolDefinition` 如何进入 `AgentSession` runtime?
- 内置工具有没有单独设置 `executionMode`?

## Ground Truth

`ToolName` 的字面量 union 是 `"read" | "bash" | "edit" | "write" | "grep" | "find" | "ls"` [E: packages/coding-agent/src/core/tools/index.ts:83]。`allToolNames` 是同一组七个 wire name 的 `Set<ToolName>` [E: packages/coding-agent/src/core/tools/index.ts:84]。`createCodingToolDefinitions` 只返回 `read`、`bash`、`edit`、`write` 的 definition [E: packages/coding-agent/src/core/tools/index.ts:138] [E: packages/coding-agent/src/core/tools/index.ts:140] [E: packages/coding-agent/src/core/tools/index.ts:141] [E: packages/coding-agent/src/core/tools/index.ts:142] [E: packages/coding-agent/src/core/tools/index.ts:143]。`createReadOnlyToolDefinitions` 只返回 `read`、`grep`、`find`、`ls` 的 definition [E: packages/coding-agent/src/core/tools/index.ts:147] [E: packages/coding-agent/src/core/tools/index.ts:149] [E: packages/coding-agent/src/core/tools/index.ts:150] [E: packages/coding-agent/src/core/tools/index.ts:151] [E: packages/coding-agent/src/core/tools/index.ts:152]。`createAllToolDefinitions` 返回以 `ToolName` 为 key 的 record,逐一装入全部七个 definition [E: packages/coding-agent/src/core/tools/index.ts:156] [E: packages/coding-agent/src/core/tools/index.ts:158] [E: packages/coding-agent/src/core/tools/index.ts:159] [E: packages/coding-agent/src/core/tools/index.ts:160] [E: packages/coding-agent/src/core/tools/index.ts:161] [E: packages/coding-agent/src/core/tools/index.ts:162] [E: packages/coding-agent/src/core/tools/index.ts:163] [E: packages/coding-agent/src/core/tools/index.ts:164]。

`ToolDefinition.executionMode` 是可选字段 [E: packages/coding-agent/src/core/extensions/types.ts:461]。`wrapToolDefinition` 会把 definition 的 `executionMode` 原样转给 core `AgentTool` [E: packages/coding-agent/src/core/tools/tool-definition-wrapper.ts:15]。core `AgentTool.executionMode` 也是可选字段 [E: packages/agent/src/types.ts:393],`Agent` 构造时默认 `toolExecution` 为 `"parallel"` [E: packages/agent/src/agent.ts:218]。因此本目录把内置工具的 `executionMode` 记为 `unset -> default parallel`,因为七个 `createXToolDefinition` 返回对象都未声明 `executionMode` [I]。
