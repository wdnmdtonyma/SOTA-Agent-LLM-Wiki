> `InteractiveMode` 是 pi-coding-agent 的 long-lived TUI host: 它把 terminal rendering、editor input、extension UI、session lifecycle rebinding 和 `AgentSession.prompt()` loop 接在同一个交互式事件编排层里。

## 能回答的问题

- `InteractiveMode.run()` 怎样把用户输入送进 `AgentSession.prompt()`?
- TUI 页面里 header、resources、chat、pending、status、widgets、editor、footer 是在哪里装配的?
- slash command、bash command、streaming steer、follow-up 和普通 prompt 的分派边界是什么?
- `AgentSessionEvent` 怎样变成 assistant message、tool execution、compaction、retry 和 footer 更新?
- startup selector / input / first-time setup 为什么不直接复用 `InteractiveMode`?
- extension 如何替换 editor、header、footer、widget 或打开 selector/input/editor dialog?
