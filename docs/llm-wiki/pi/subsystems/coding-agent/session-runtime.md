> `AgentSessionRuntime` 是 pi-coding-agent 在一个 host 中持有“当前 `AgentSession` + 当前 cwd-bound services”的 lifecycle owner: `/new`、resume、fork、import 和 quit 都通过它先通知 extension、再 teardown 旧 session、再用同一个 factory 创建 replacement runtime。

## 能回答的问题

- `createAgentSessionRuntime()` 和 `AgentSessionRuntime` 分别负责什么?
- `/new`、resume、fork、import 为什么会重新创建 cwd-bound services?
- session replacement 时 extension 事件的顺序是什么, 哪些事件可以取消?
- `withSession` 和 `setRebindSession()` 解决哪类 stale context 问题?
- persisted session、in-memory session 和 import JSONL 在 runtime 层有什么差异?
- 缺失 session cwd 是在哪里阻断的?

## 设计动机与权衡

`AgentSessionRuntime` 把 “current session pointer” 和 “target cwd-bound services” 捆在一起, 避免 resume 到另一个 cwd 后继续使用旧 `SettingsManager`、`ModelRegistry` 或 `ResourceLoader` [E: packages/coding-agent/src/core/agent-session-runtime.ts:177] [E: packages/coding-agent/src/core/agent-session-runtime.ts:179] [I: packages/coding-agent/src/core/agent-session-services.ts:69] [I: packages/coding-agent/src/core/agent-session-services.ts:74] [I]。`AgentSessionServices` 的 service creation 独立于 session creation, 让 caller 可以先拿目标 cwd 的 services 再解析 model、thinking、tools 和其他 session inputs [I: packages/coding-agent/src/core/agent-session-services.ts:49] [I: packages/coding-agent/src/core/agent-session-services.ts:55] [I: packages/coding-agent/src/core/agent-session-services.ts:180] [I: packages/coding-agent/src/core/agent-session-services.ts:187]。

replacement 事件采用 “before event 可取消, shutdown 不取消, start metadata 交给新 session” 的形状: `emitBeforeSwitch()` / `emitBeforeFork()` 只在 handler 存在时发 event 并读取 `cancel`;`teardownCurrent()` 总是发 `session_shutdown`;replacement creation 传入 `sessionStartEvent` 记录 reason 和 previous session file [E: packages/coding-agent/src/core/agent-session-runtime.ts:133] [E: packages/coding-agent/src/core/agent-session-runtime.ts:142] [E: packages/coding-agent/src/core/agent-session-runtime.ts:147] [E: packages/coding-agent/src/core/agent-session-runtime.ts:150] [E: packages/coding-agent/src/core/agent-session-runtime.ts:159] [E: packages/coding-agent/src/core/agent-session-runtime.ts:164] [E: packages/coding-agent/src/core/agent-session-runtime.ts:167] [E: packages/coding-agent/src/core/agent-session-runtime.ts:248]。这给 extension 一个最后拦截点, 同时让已确认的 replacement teardown 路径保持单向 [I]。
