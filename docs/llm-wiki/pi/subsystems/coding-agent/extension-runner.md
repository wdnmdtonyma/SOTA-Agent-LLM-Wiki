> `ExtensionRunner` 是 pi-coding-agent 的扩展执行引擎: 它把 loader 产出的 extensions 和 shared runtime 绑定到当前会话, 构造 `ExtensionContext`, 并按事件类型顺序调用 extension handlers。

## 能回答的问题

- `bindCore()` 具体把哪些 session action 和 context action 绑定进 extension runtime?
- `ExtensionRunner.createContext()` 给 event handler 和 extension tool 暴露哪些运行时能力?
- 通用 `emit()` 和 `emitToolCall()`、`emitInput()` 等专用 emitter 的短路/聚合规则有什么不同?
- extension 注册的 tools、commands、shortcuts、flags、message renderers 如何在 runner 层汇总?
- project trust、provider registration、session-before cancellation 这些特殊路径怎样处理 error 和 result?
- reload/session replacement 后旧 extension context 为什么会变 stale?
