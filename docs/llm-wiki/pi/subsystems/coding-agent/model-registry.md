> `model-registry` 是 pi-coding-agent 的产品层 model registry: `ModelRegistry` 把 `pi-ai` built-in model catalog、用户 `models.json`、extension 动态 provider、`AuthStorage` credential 和 request headers 合成一个可供 CLI/session 查找与请求鉴权的模型视图。

## 能回答的问题

- `ModelRegistry` 从哪里加载 built-in models 和 custom models?

## Gotcha

- `models.json` override-only provider entries are valid only when they set at least one of `baseUrl`, `headers`, `compat`, `modelOverrides`, or `models`; an empty provider entry is rejected [E: packages/coding-agent/src/core/model-registry.ts:541] [E: packages/coding-agent/src/core/model-registry.ts:543] [E: packages/coding-agent/src/core/model-registry.ts:545]。
- A non-built-in provider with custom models must specify `baseUrl`; built-in providers with custom models may inherit `api` and `baseUrl` from built-in defaults [E: packages/coding-agent/src/core/model-registry.ts:548] [E: packages/coding-agent/src/core/model-registry.ts:551] [E: packages/coding-agent/src/core/model-registry.ts:580] [E: packages/coding-agent/src/core/model-registry.ts:587] [E: packages/coding-agent/src/core/model-registry.ts:589] [E: packages/coding-agent/src/core/model-registry.ts:601] [E: packages/coding-agent/src/core/model-registry.ts:604]。
- `getApiKeyAndHeaders()` calls `AuthStorage.getApiKey()` with `includeFallback: false`, so generic environment fallback from `AuthStorage` is not used at this step; provider-level `apiKey` templates in `models.json` can still resolve environment variables through `resolveConfigValueOrThrow()` [E: packages/coding-agent/src/core/model-registry.ts:705] [E: packages/coding-agent/src/core/auth-storage.ts:513] [E: packages/coding-agent/src/core/auth-storage.ts:516] [E: packages/coding-agent/src/core/model-registry.ts:709] [E: packages/coding-agent/src/core/resolve-config-value.ts:229]。
- `getProviderAuthStatus()` reports `runtime` source from `AuthStorage` as `configured: false` even when `runtimeOverrides` has a CLI key; consumers appear to use `source`/`label` for status display, but whether that boolean is intentional is not established in this node [E: packages/coding-agent/src/core/auth-storage.ts:359] [E: packages/coding-agent/src/core/auth-storage.ts:360] [U]。
- `authHeader: true` copies an API key into `Authorization` but still returns `apiKey` in the result; downstream callers must understand whether the wire API wants the key as an option, a header, or both [E: packages/coding-agent/src/core/model-registry.ts:732] [E: packages/coding-agent/src/core/model-registry.ts:736] [E: packages/coding-agent/src/core/model-registry.ts:741] [I]。
