> `subsys.ai.google-vertex` 描述 `pi-ai` 的 `google-vertex` wire 协议入口:它用 `@google/genai` 的 Vertex 模式构造客户端,把 pi 的 `Context` 转成 Gemini `GenerateContentParameters`,并把 Vertex/Gemini stream chunk 归一成 `AssistantMessageEventStream`。

## 职责边界

`GoogleVertexOptions` 是 `StreamOptions` 的 Vertex 扩展,它保留 `toolChoice` 与 `thinking` 配置,并额外接受 `project` 与 `location` 两个 Vertex 定位字段。[E: packages/ai/src/api/google-vertex.ts:43][E: packages/ai/src/api/google-vertex.ts:50][E: packages/ai/src/api/google-vertex.ts:51] `stream` 的 API identity 固定为 `"google-vertex"`,输出消息也写入 `api: "google-vertex"` 与当前 `model.provider` / `model.id`。[E: packages/ai/src/api/google-vertex.ts:68][E: packages/ai/src/api/google-vertex.ts:79][E: packages/ai/src/api/google-vertex.ts:80][E: packages/ai/src/api/google-vertex.ts:81]

`stream` 不是 provider registry 或 auth discovery 层;它假定外层已经把 provider/model/options 传入,然后在 wire module 内部解析 Vertex API key、ADC project/location、payload 与 streaming events。[I] API key 如何从环境发现不在本节点 source 范围内;本节点只解释 `api/google-vertex.ts` 在真正发请求前如何消费 `options.apiKey`、`options.env`、`project` 与 `location`。[E: packages/ai/src/api/google-vertex.ts:95][E: packages/ai/src/api/google-vertex.ts:97][E: packages/ai/src/api/google-vertex.ts:99][E: packages/ai/src/api/google-vertex.ts:409][E: packages/ai/src/api/google-vertex.ts:421][E: packages/ai/src/api/google-vertex.ts:434]

- A usable `options.apiKey` selects API-key client construction and bypasses `resolveProject` / `resolveLocation`; ADC mode calls both resolvers, and each throws when its value is unavailable.[E: packages/ai/src/api/google-vertex.ts:95][E: packages/ai/src/api/google-vertex.ts:97][E: packages/ai/src/api/google-vertex.ts:98][E: packages/ai/src/api/google-vertex.ts:99][E: packages/ai/src/api/google-vertex.ts:421][E: packages/ai/src/api/google-vertex.ts:426][E: packages/ai/src/api/google-vertex.ts:427][E: packages/ai/src/api/google-vertex.ts:434][E: packages/ai/src/api/google-vertex.ts:436][E: packages/ai/src/api/google-vertex.ts:437]
- `resolveCustomBaseUrl` ignores base URLs containing `{location}` instead of templating them; callers that expect runtime interpolation need to provide a concrete base URL or rely on the SDK's project/location endpoint construction.[E: packages/ai/src/api/google-vertex.ts:387][E: packages/ai/src/api/google-vertex.ts:389][E: packages/ai/src/api/google-vertex.ts:390][I]
- Vertex consumes `candidate.content.parts` from `client.models.generateContentStream(params)` chunks before mapping text/function-call parts into pi events; backend-level equivalence with Gemini API is outside this node's evidence scope.[E: packages/ai/src/api/google-vertex.ts:105][E: packages/ai/src/api/google-vertex.ts:115][E: packages/ai/src/api/google-vertex.ts:116][I]
